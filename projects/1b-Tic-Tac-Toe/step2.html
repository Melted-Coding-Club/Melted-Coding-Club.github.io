Project 1b: Tic Tac Toe

Step 2 - Handling User Input and Player Turns

  • We start by defining the current player (line10). For this project, we’ll use 0 for "x" and 1 for "o." This makes it easier to manage player turns.
  • We also create a list called player_symbols (line 11) to associate each player with their respective symbol ("x" or "o"). player_symbols[0] will be "x" and player_symbols[1] will be "o."
  • Next (line 12), we print the empty board to show the current state of the game.
  • We then prompt the user to choose a square on the board by entering a number (1-9) (line 14). If the chosen square is empty (line 16), we fill it with the current player’s symbol (line 17). If the square is already filled (line 18), we print a message indicating that it’s taken (line 19).
  • Finally, we print the updated board to reflect the player’s move (line 20).
